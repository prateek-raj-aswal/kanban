Project Brief

Overview REQUIRED

Kanban is a collaborative project management platform for individuals and small-to-medium software teams. It provides visual Kanban boards, issue tracking, real-time collaboration, and flexible workflows — positioned as a lightweight Jira alternative that avoids enterprise complexity while remaining scalable. Built with a Java Spring Boot backend, Next.js frontend, PostgreSQL, and WebSocket real-time sync.

Goals

  • Primary: Ship a fully functional MVP (auth, workspaces, projects, Kanban boards, drag-and-drop tasks, issues, real-time) by 2026-06-30.
  • Secondary: Beta release with comments, notifications, advanced filtering by 2026-08-15.
  • GA: Production-ready with RBAC, observability, and security hardening by 2026-10-01.
  • Anti-goal: No AI sprint planning, no Gantt charts, no native mobile apps, no plugin marketplace, no time tracking/billing in v1.

Non-Functional Requirements

  • Performance: API P50 < 100ms, P95 < 500ms, P99 < 2s; board interaction latency < 100ms; real-time sync delay < 500ms; page load < 2s.
  • Security: OAuth 2.0 + JWT; RBAC (Owner/Admin/Member/Viewer); AES-256 at rest; TLS 1.2+; Trivy/Snyk in CI; OWASP Top 10 compliance.
  • Observability: Structured JSON logs mandatory; Prometheus + Grafana metrics; health endpoints on all services; distributed tracing planned.
  • Accessibility: WCAG 2.1 Level AA target for all public pages.
  • Browser support: Chrome/Firefox/Edge/Safari latest 2 versions; responsive web mandatory.

Key Decisions Already Made

  • PostgreSQL as primary DB — not negotiable.
  • All API routes versioned under /api/v1/.
  • Java Spring Boot backend — not negotiable.
  • Next.js (App Router, client-rendered) frontend — not negotiable.
  • No paid UI component libraries.
  • Flyway for migrations (currently at V32); next migration is V33.
  • Short-lived JWT access tokens + rotating refresh tokens (refresh_tokens table in PostgreSQL, Redis-swappable later).
  • RBAC with roles: Owner, Admin, Member, Viewer — enforced via central policy per workspace and per board.
  • Readable IDs (PREFIX-NNN format: US-NNN, FEAT-NNN, BUG-NNN) assigned to all cards and issues at creation via workspace-scoped atomic counter (workspace_id_counters table, SELECT FOR UPDATE).
  • Card color (hex) and column header color supported; ColorPalette is a shared UI component.
  • Modules/tags (board-scoped), swimlane group-by (none/assignee/priority/module), and issues as child entities of story-cards are all implemented.

Out of Scope (for current phase)

  • AI sprint planning and AI-powered task summarization.
  • Gantt charts and calendar view.
  • Native mobile apps (iOS/Android) — web-first.
  • Plugin/marketplace ecosystem.
  • Time tracking and billing.
  • Video/audio calls.
  • Advanced analytics dashboards.
  • Multi-region deployments and multi-tenant enterprise isolation.
  • GitHub/GitLab and Slack/Discord integrations.
  • Workflow automation engine.
  • Sprint planning and burndown charts.
  • Redis caching layer (PostgreSQL-backed until traffic warrants it).
