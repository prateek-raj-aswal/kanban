Docker Desktop — Local Development Context

Kanban runs entirely via Docker Compose. The compose files are at the repo root. All services are containerized; local development uses docker-compose.dev.yml as an override for hot-reload mounts.

Prerequisite: Docker Desktop must be running before executing any docker or docker compose commands. On Windows 11, Docker Desktop uses WSL2 backend.

E2E Browser Testing

# 1. Start all services
docker compose up -d

# 2. Wait for services to be healthy
docker compose ps

# 3. Run Playwright E2E tests (from codebase/frontend)
cd codebase/frontend
npx playwright test

Database Migrations

# Flyway runs automatically on backend startup.
# To run manually:
docker compose exec backend ./gradlew flywayMigrate

# To check migration status:
docker compose exec backend ./gradlew flywayInfo

# Current migration version: V32
# Next migration must be: V33__<description>.sql
