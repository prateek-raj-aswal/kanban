Constraints

Hard constraints the architect agent reads before producing the system design. These override suggestions that would otherwise be technically valid. Leave sections empty if no constraints apply; do not delete them.

Technical Constraints

AreaConstraintReason
e.g. Database Must use PostgreSQL — no migration to another DB allowed Existing production data
e.g. API versioning All routes under /api/v1/ — prefix is contractual Existing clients depend on it
e.g. Auth Must integrate with existing Auth0 tenant — no custom auth Org-wide SSO policy

Dependency Constraints

Package / ServiceConstraintReason
e.g. React Must stay on v18 — no upgrade to v19 yet Dependency on library X not yet compatible
e.g. Node.js v22 LTS only — matches CI image CI/CD pinned image

Security Constraints

  • e.g. No PII may be written to application logs.
  • e.g. All endpoints except /health and /auth/login require a valid JWT.
  • e.g. Passwords must be hashed with bcrypt, cost factor ≥ 12.
  • e.g. CORS must be restricted to specific origins — no wildcard in production.
